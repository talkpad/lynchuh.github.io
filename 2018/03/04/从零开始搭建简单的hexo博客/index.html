GitHub的基础使用

使用github前最好先翻下下下墙，在墙内github的反应真的贼慢。

如何在本地使用命令行操作github：

配置GitHub，添加SSH钥匙

Step 1
打开Git Bash，运行命令$ ssh-keygen -t rsa -b 4096 -C "your_email@example.com",记得填写自己的邮箱地址。
Step 2
依次按回车3次。
Step 3
获取公钥：运行命令cat ~/.ssh/id_rsa.pub.此处不要忘记后缀.pub，有后缀才是公钥文件。
Step 4
在GitHub上添加钥匙，路径：Settingss→SSH and GPG keys→New SSH key。填写title和key（将step3获取的信息copy下来）。

在这里，关于git的使用，个人理解在本地应该有三个区->
本地目录：你新建的文件都在这个目录里。->
暂存区：本地目录的文件改动后需要add到暂存区。->
本地仓库：将暂存区的文件提交到仓库需commit。只有本地仓库的内容才可以push到远程仓库。

搭建博客的环境准备

git

有了git命令才可以将博客部署到github上。 记得部署git，步骤查看1.1

node.js

hexo基于node.js，下载hexo前需安装Node.js.安装结束后直接在Git Bash中配置npm(不要打开node.js)

Hexo

hexo是我们个人博客网站的框架，安装hexo框架需与博客放在同一个文件夹内。（安装命令在3.2）

使用hexo+github搭建一个个人博客

建立远程仓库

打开github，新建一个空的仓库（repository），名字为【用户名.github.io】,注：用户名是github上的username

建立本地仓库目录，安装博客框架

本地修改博客配置

在blog的根目录有一个文件_config.yml，这是站点配置文件。运行命令start _config.yml打开文件，修改配置：

  • title —- 网站标题
  • subtitle —- 网站副标题
  • description —- 网站描述
  • author —- 网站作者
  • Repo —- 远程仓库地址（复制3.1里建立的远程仓库的SSH地址后粘贴）
    需要注意的是 在配置的时候需要在冒号后面加一个英文空格

    部署到GitHub

到此博客已经基本部署完成。打开github上的远程仓库【用户名.github.io】的setting上 Github pages，可以查看到博客预览链接。

主题修改

小白想自己做一个主题还是别妄想了，在github上有hexo主题的合集

  • 找到合适的主题后，进入该主题的github仓库，复制它的SSH地址
  • 下载该主题仓库到本地：
    进入到blog的themes目录，运行命令git clone 主题SSH地址,下载喜欢的主题库到本地。->运行命令cd ..返回blog目录，再运行start _config.yml打开站点配置文件，修改配置内的themes属性。

发表博客文章

我的文件路径是：myblog/source/_posts/xxx.md

上传博客源码到github仓库上

在3.1中建立的仓库【用户名.github.io】保存的只是博客，并没有保存博客的源代码，再建立一个仓库【blog-generator】用于保存源代码可防止因为巫山blog而丢失所有数据。->
注意，每次部署（hexo deploy）完后记得更新源码仓库(add→commit→pull→push)->
具体步骤查看1.2
