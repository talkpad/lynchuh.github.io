从零开始搭建简单的hexo博客

Posted on 2018-03-04

回想了以下，刚对前端这个行业有所了解应该是在17年的10月份，那时是公司的程序猿小哥提到的，后来在知乎里查了一下，才逐渐有所了解。刚开始的自学是在freecodecamp里边闯关边查资料。但是到后面挑战越来越难，才知道自己的知识面真的很窄，仅靠html、css和js这点知识不足以充当一名合格的前端程序员。->
来到饥人谷，才发现，程序员的世界很广阔，额。前路漫漫..

以下，这是一个全新手的从零开始搭建hexo博客的总结。新手小白可参考，如有不对的地方，烦请大佬交流指正，谢谢！

1. 搭建博客需要的背景知识

1.1 命令行

我是在windows环境下使用git bash的，首先需区分git bash与git的区别：->
Git bash是一个给我们提供虚拟的linux环境的软件，而git只是其中的一个命令。一般git bash已内置了git命令，无需另外安装下载，但是在使用git命令前需配置。->
//配置命令
$ git config –global user.name xxx
$ git config –global user.email yyy
$ git config –global push.default simple
$ git config –global core.quotepath false
$ git config –global core.editor “vim”

基本的命令行操作

1.1.1 创建命令

$ mkdir xxx //新建目录
$ mkdir -p  //新建目录路径
$ touch xxx.xxx  //新建文件（若文件已存在，使用可改变文件更新时间）
$ echo "内容">文件路径   //新建文件

延伸
$ echo “内容”>!文件路径 //强制创建文件（将原文件的内容全部更改）
$ echo “内容 “>>文件路径 //追加文件内容

1.1.2 进入/打开命令

$ cd 文件路径  //进入某个目录
$ start 文件路径 //打开某个文件
$ vim 文件路径 //使用编辑器vim打开某个文件

1.1.3 复制命令

$ cp 源路径 目标路径  //复制文件
$ cp -p  源路径 目标路径  //复制目录

1.1.4 删除命令

$ rm -rf  文件路径/目录路径 //-r 递归地删除 -f 无需经过同意删除文件

1.1.5 查看信息

$ pwd   //显示当前目录
$ whoami  //查看用户名
$ ls (-al)  路径  //查看路径 （-a：所有文件，包括隐藏文件 -l：使用列表展示文件信息）
$ tree  //查看目录结构

1.2 GitHub的基础使用

使用github前最好先翻下下下墙，在墙内github的反应真的贼慢。

如何在本地使用命令行操作github：

1.2.1 配置GitHub，添加SSH钥匙

Step 1
打开Git Bash，运行命令$ ssh-keygen -t rsa -b 4096 -C "your_email@example.com",记得填写自己的邮箱地址。
Step 2
依次按回车3次。
Step 3
获取公钥：运行命令cat ~/.ssh/id_rsa.pub.此处不要忘记后缀.pub，有后缀才是公钥文件。
Step 4
在GitHub上添加钥匙，路径：Settingss→SSH and GPG keys→New SSH key。填写title和key（将step3获取的信息copy下来）。

1.2.2 关联远程仓库和本地仓库

在github上新建一个空的仓库后，地址选取SSH。接下来在本地操作。->
新建目录后，在该目录下运行Git Bash

  • 本地为空仓库
    复制空仓库下..or create a new repository on the command line 下的命令。
    eg：
    $ echo "# xxx" >> README.md
$ git init
$ git add README.md
$ git commit -m "first commit"
$ git remote add origin git@github.com:用户名/xxx.git
$ git push -u origin master
  • 本地仓库已有文件
    复制空仓库下 ..or push an existing repository from the command line 下的命令
    eg：
    $ git remote add origin git@github.com:用户名/xxx.git
$ git push -u origin master

1.2.3 本地文件变动后上传更新到GitHub

在本地仓库运行git bash ，某些文件进行编辑后，信息已变动，若要上传github，则在该目录下运行以下命令：
$ git add . //将当前目录所有文件添加到暂存区
$ git commit -m “信息” //将暂存区的文件正式提交到本地仓库（.git）
$ git pull
$ git push

在这里，关于git的使用，个人理解在本地应该有三个区->
本地目录：你新建的文件都在这个目录里。->
暂存区：本地目录的文件改动后需要add到暂存区。->
本地仓库：将暂存区的文件提交到仓库需commit。只有本地仓库的内容才可以push到远程仓库。

另外：
$ git status -sb //显示当前所有文件的状态
$ git log //查看变更历史

1.2.4 下载github上的仓库到本地

打开要下载的仓库，复制该仓库的SSH地址，在需要下载的目录内运行以下命令：
$ git clone 仓库的SSH地址

1.3 markdown语法

之前我并没有接触过markdown书法，因此我是先搭建好了博客，然后google了解了基本语法后再写文章的。

2. 搭建博客的环境准备

2.1 git

有了git命令才可以将博客部署到github上。 记得部署git，步骤查看1.1

2.2 node.js

hexo基于node.js，下载hexo前需安装Node.js.安装结束后直接在Git Bash中配置npm(不要打开node.js)

2.3 Hexo

hexo是我们个人博客网站的框架，安装hexo框架需与博客放在同一个文件夹内。（安装命令在3.2）

3. 使用hexo+github搭建一个个人博客

3.1 建立远程仓库

打开github，新建一个空的仓库（repository），名字为【用户名.github.io】,注：用户名是github上的username

3.2 建立本地仓库目录，安装博客框架

$ cd 路径 //不要选取系统根目录为本地仓库目录 
$ npm install -g hexo-cli   //安装hexo
$ Hexo init  blog  //初始化博客
$ npm i    //我也不懂

3.3 本地修改博客配置

在blog的根目录有一个文件_config.yml，这是站点配置文件。运行命令start _config.yml打开文件，修改配置：

  • title —- 网站标题
  • subtitle —- 网站副标题
  • description —- 网站描述
  • author —- 网站作者
  • Repo —- 远程仓库地址（复制3.1里建立的远程仓库的SSH地址后粘贴）
    需要注意的是 在配置的时候需要在冒号后面加一个英文空格

    3.4 部署到GitHub

    $ npm install hexo-deployer-git –save //安装git部署插件
    $ hexo deploy //部署到GitHub

到此博客已经基本部署完成。打开github上的远程仓库【用户名.github.io】的setting上 Github pages，可以查看到博客预览链接。

3.5 主题修改

小白想自己做一个主题还是别妄想了，在github上有hexo主题的合集

  • 找到合适的主题后，进入该主题的github仓库，复制它的SSH地址
  • 下载该主题仓库到本地：
    进入到blog的themes目录，运行命令git clone 主题SSH地址,下载喜欢的主题库到本地。->运行命令cd ..返回blog目录，再运行start _config.yml打开站点配置文件，修改配置内的themes属性。
    $ theme（第75行）：主题名字 //不知道的可以打开文件目录的themes里查看
    $ hexo generate //生成静态文件
    $ hexo deploy //部署到github

    3.6 发表博客文章

    $ hexo new 博客题目
    $ start 文件路径 //打开文件后使用markdown语法编辑内容。
    $ hexo generate
    $ hexo deploy
    我的文件路径是：myblog/source/_posts/xxx.md

4. 上传博客源码到github仓库上

3.1中建立的仓库【用户名.github.io】保存的只是博客，并没有保存博客的源代码，再建立一个仓库【blog-generator】用于保存源代码可防止因为巫山blog而丢失所有数据。->
注意，每次部署（hexo deploy）完后记得更新源码仓库(add→commit→pull→push)->
具体步骤查看1.2
